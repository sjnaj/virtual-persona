🏗️ 系统架构总览

她是唯一的个体——一个大脑、一份情绪、一段人生。但她在不同聊天窗口面对不同的人时，会自然地切换语气和边界，就像真人在微信里和不同的人聊天一样。

💡 核心设计原则：紫色区域是她的"大脑"，全局唯一；粉色区域是她对每个人的"印象"，各自独立；蓝色区域是跨场景共享的智能，但带有隐私标记。

🧩 模块详解

每个模块都是可独立理解的组件。点击查看详情。

🎭

Orchestrator 中枢协调器

整个系统的指挥中心。接收所有外部消息，协调各模块协同工作，管理后台循环任务。它知道"她"在做什么、跟谁说话、该不该回。

orchestrator.py
🌙

LifeSimulator 生活模拟

模拟她的24小时日常：起床、上班、吃饭、下班、刷手机、睡觉。包括体力、饥饿度、位置信息。影响她的可用性和回复速度。

life_simulator.py
💛

EmotionEngine 情绪引擎

PAD情绪模型（愉悦度-激活度-支配度）+ 依恋系统。情绪会影响她说话的语气、回复速度、表情包选择，甚至她是否想主动找人聊天。

emotion_engine.py
👥

RelationshipManager 关系系统

每个人在她心里都有一份独立的"印象档案"：亲密度、信任度、标签、常聊话题、雷区、inside jokes。关系会随互动自然演化。

relationship.py
🧠

MemoryHub 记忆中枢

三层记忆：情节记忆、语义记忆、情感印记。记忆带隐私标记（来自私聊还是群聊）。支持自然遗忘、模糊化和情感加权。

memory_hub.py
💭

ChatContextManager 上下文管理

每个聊天窗口独立的对话流管理。维护最近消息历史、会话活跃状态、群成员列表。区分私聊和群聊的上下文。

chat_context.py
🎯

GroupBehaviorEngine 群聊行为

决定她在群里何时说话、怎么说。考虑发送者亲密度、话题匹配、情绪、性格外向性、连续沉默时长等因素。模拟真人的群聊行为模式。

group_behavior.py
✍️

ExpressionSynthesizer 表达合成

最终输出层。综合情绪、关系、记忆、上下文、生活状态，生成人格化的消息。根据亲密度调整语气，群聊中自动克制。

expression.py
📱

BrowserAgent 浏览代理

模拟她刷手机：RSS、微博热搜、感兴趣的内容。浏览到的内容可能在聊天中自然提起，但不会强行安利。

browser_agent.py
😸

StickerEngine 表情包引擎

表情包的收集、分类和使用。根据心情和语境选择表情包。表情包是她个性的重要体现。

sticker_engine.py
💬

ProactiveEngine 主动引擎

触发主动发消息的时机判断：看到有趣内容想分享、想念某人、到了特定时间等。选择最合适的人发送。

proactive_engine.py
🔌

EventBus 事件总线

模块间的发布/订阅通信机制。解耦各模块的依赖关系。情绪变化、生活事件、用户消息等通过事件广播。

event_bus.py

🔄 数据流

一条消息从收到到回复的完整旅程。

STEP 1 — 触发评估

ProactiveEngine.evaluate()

定期（每5分钟）检查是否有主动发消息的触发条件。来源包括：看到有趣内容想分享、想念某人（依恋度高 + 久未联系）、到了特定时间点（早安/晚安）、情绪需要倾诉。

STEP 2 — 选择目标

选择发给谁

按亲密度排序，选择最适合的对象。比如想分享音乐相关内容→选常聊音乐的朋友；想倾诉→选最亲近的人。不会给不太熟的人突然发消息。

STEP 3 — 生成消息

Expression.compose_proactive()

和普通回复的生成流程相同，但 prompt 中标明是"主动想说"而非"回复对方"。内容更随意：分享链接、吐槽、日常碎碎念。

STEP 4 — 发送

通过已注册的回调发送

orchestrator 为每个聊天窗口注册了 proactive_callback。触发后直接发送到对应的 Telegram 聊天。

## 5个后台循环任务，持续运行 Loop 1: _life_tick_loop # 每15分钟 → LifeSimulator.tick() # 更新作息/体力/位置 → EmotionEngine.passive_decay() # 情绪自然衰减 Loop 2: _browse_loop # 每60分钟 → BrowserAgent.should_browse() # 判断是否要刷手机 → BrowserAgent.browse() # 抓取RSS/热搜 → EventBus → emotion_change # 浏览内容影响情绪 Loop 3: _proactive_loop # 每5分钟 → ProactiveEngine.evaluate() # 是否主动发消息 → → 选择目标 → Expression → 发送 Loop 4: _memory_loop # 每6小时 → MemoryHub.consolidate() # 缓冲区→长期记忆 → MemoryHub.forget_and_distort() # 自然遗忘+扭曲 Loop 5: _conversation_end_check # 每2分钟 → 检测30分钟无互动的窗口 # 标记会话结束 → 触发该窗口的记忆整合 # 及时巩固

👥 群聊引擎详解

真人在群里不是每条消息都回的。她会潜水、偶尔冒泡、被@了才认真回、和熟人互动更多。

回复概率计算

# 基础概率从0开始，逐项叠加 base_prob = 0.0 # 1. 被@了 → 100%回复（硬规则） if mentioned_me: return 1.0 # 2. 发送者亲密度 base_prob += closeness × 0.25 # 好朋友(0.8)→+0.20, 陌生人(0.1)→+0.02 # 3. 话题匹配兴趣 base_prob += 0.15 # 每匹配一个兴趣关键词 # 4. 性格外向性 base_prob += extraversion × 0.08 # 外向的人更爱在群里说话 # 5. 情绪影响 if mood_good: base_prob += 0.08 if mood_bad: base_prob -= 0.10 # 6. 连续沉默修正（不能一直潜水） if silence > 15: base_prob += 0.10 if silence > 30: base_prob += 0.20 # 7. 刚说过话（防刷屏） if 刚说完 < 2min: base_prob ×= 0.20 # 8. 有人接自己话 → +0.15 # 9. 消息含问号/邀请 → +0.08 # 最终 prob ∈ [0, 0.85]

三种回复模式

🎯 Direct

正经回复消息内容。多条消息、表情包都可能出现。适用于被@、话题高度相关、和亲近的人互动。

😂 Reaction

轻回应——"哈哈"、"+1"、一个表情包、一句简短感叹。适用于概率偏低但还是想参与的场景。

🔗 Piggyback

借话题展开自己想说的。看到别人聊某个话题，顺势分享自己的经历或看法。很自然的群聊行为。

🔐 隐私边界系统

她知道什么话是在哪里听到的，不会在群里说出别人私聊里的秘密。

🔓 Public 记忆

来自群聊或公开场合的信息。可以在任何场景中自然提起。

"A在群里说他最近在学画画" → source_privacy: "public" → 可以在任何场景提起

🔒 Private 记忆

来自私聊的信息。在群聊中检索到时，标记为 private_hints——"你知道但不该说"。

"A私聊跟我说他要辞职" → source_privacy: "private" → 群里有人问A近况时： prompt: "⚠️ 你知道但不该说" 她的回复: "我也不太清楚诶"
⚠️ 隐私系统的效果取决于LLM的遵从能力。在prompt中明确标注"不要在群里说出来"后，大模型通常能很好地遵守。但建议在高敏感场景中额外做关键词过滤检测。

启动后的初始化流程

STEP 1

启动 Bot，获取 bot_username

系统启动后自动获取 bot 的用户名，用于群聊中的 @检测。

STEP 2

加载已有的关系和记忆

从 data/relationships.json 和 data/chroma/ 加载持久化数据。第一次启动为空。

STEP 3

启动后台循环

5个异步循环任务开始运行（生活模拟、浏览、主动消息、记忆整合、会话检测）。

STEP 4

设定初始关系（可选）

用管理员命令 /set_role 和 /inject_memory 给她植入"前置记忆"和"已有关系"。

STEP 5

开始正常互动

用户发消息，或者把bot拉进群聊。关系从这里开始自然发展。

📖 场景手册

常见运维场景的操作指南。点击展开查看详情。

1

让她认识一个新朋友

▼

你想让你的朋友也能跟她聊天，从"陌生网友"开始慢慢变熟。

# 1. 获取朋友的 Telegram user_id（让他先私聊bot一次，在日志里看） # 2. 加入白名单 /allow_user 987654321 # 3. 可选：预设关系起点 /set_role 987654321 网友 0.3 # 4. 可选：注入前置记忆 /inject_memory 在一个群里认识了一个叫小明的人，他好像是做前端开发的，挺有趣的 # 5. 让你朋友去私聊 bot，从此自然发展
💡 如果不预设，她会把这个人当完全的陌生人对待——礼貌但冷淡。预设 closeness=0.3 + 注入一段共同记忆，可以让关系有个更自然的起点。
2

把她拉进一个群聊

▼

让她在群里和多个人自然互动。

# 1. 在 Telegram 中把 bot 拉入群聊 # 确保 bot 有"读取消息"权限（关闭Privacy Mode） # @BotFather → /mybots → 选bot → Group Privacy → Disable # 2. 管理员在群里发送： /allow_group # 3. 她开始"潜水"，观察群消息 # 4. @她 时一定会回复 # 5. 看到感兴趣的话题可能会随机冒泡 # 6. 和群里她认识的人互动更频繁
⚠️ 重要：必须在 @BotFather 中关闭 Privacy Mode，否则 bot 在群里只能看到 @它 的消息和命令，无法实现潜水+随机参与。
3

让两个AI在群里互动

▼

把你的AI和另一个AI（或别人的AI）放在同一个群里，让它们自然发展关系。

# 方案A：群聊互动 # 1. 把两个bot都拉进同一个群 # 2. 分别在群里 /allow_group # 3. 它们会各自看到对方的消息 # 4. 注意：可能会陷入无限对话循环！ # 建议：至少一方设置较低的群聊回复概率 # 方案B：私聊互动（需要中转脚本） # 写一个脚本作为"中间人"： # Bot-A 发消息 → 脚本接收 → 转发给 Bot-B # Bot-B 回复 → 脚本接收 → 转发给 Bot-A # 方案C：让另一个AI主动私聊她 /allow_user <other_bot_id> # 她不知道对方是AI，会自然地发展关系 # 如果对方"表现不对劲"，她可能会在标签里标注 # "说话有点奇怪" 或 "感觉像机器人"
🔁 防止无限循环的方法：在 GroupBehaviorEngine 中对 is_bot 的消息额外降低回复概率；或设置"最近5分钟内不回复同一个bot超过3次"的硬限制。
4

她守住私聊的秘密

▼

用户A在私聊中告诉她一个秘密。之后在群聊中有人提到相关话题。

# 场景： # A 私聊跟她说："我最近偷偷在准备辞职" # → 记忆入库：source_privacy = "private" # 后来群里有人问："A最近怎么样了？好久没见他发言" # → 记忆检索时命中"A准备辞职" # → 但 source_privacy = "private" + 当前 context = "group" # → 被分类为 private_hints（知道但不说） # prompt 中： "⚠️ 你从私聊中知道的（不要在群里直接说出来，但可以影响你的态度）： A好像在准备辞职" # 她的回复可能是： "我也不太清楚诶，你问他本人呀" # 或 "应该还好吧？可能最近比较忙" # 而不是："他要辞职了"
5

给她植入"前世记忆"

▼

你想让她"记得"一段从未发生的历史——比如你们是老朋友，之前一起旅行过。

# 1. 设定关系 /set_role 12345 好朋友 0.75 # 2. 注入核心记忆（多条） /inject_memory 去年夏天和他一起去了厦门旅行，在鼓浪屿吃了好多海鲜 /inject_memory 他养了一只叫"年糕"的橘猫，总是偷吃东西 /inject_memory 他以前帮我修过电脑，结果越修越坏，笑死了 /inject_memory 他最近在学吉他，弹得不太好但很认真 /inject_memory 我们有个约定，等他学会《晴天》了请我吃饭 # 3. 开始聊天时，她会自然地"记得"这些 # 但因为记忆的 confidence 会衰减， # 久了之后有些细节可能变模糊——就像真人一样
6

调整她的性格/状态

▼

想让她今天心情特别好、或者让她变得更内向。

# 性格调整：修改 config.yaml 中的 persona.personality # 然后重启（性格是静态配置，不会运行时改变） # 情绪调整（运行时）： # 目前没有直接命令，但可以通过以下方式间接影响： # 1. 在私聊中发正面/负面消息 → 触发情绪变化 # 2. 修改 emotion_engine.py 的初始值 # 3. 添加 /set_mood 命令（自行扩展） # 扩展示例：添加 /set_mood 命令 # 在 bot.py 中添加处理函数： async def _cmd_set_mood(self, update, context): valence = float(context.args[0]) # -1 ~ 1 arousal = float(context.args[1]) # 0 ~ 1 self.orch.emotion.valence = valence self.orch.emotion.arousal = arousal
7

多窗口并行对话

▼

她同时在和多个人聊天（就像你的微信一样多窗口切换）。

# 系统天然支持。每条消息独立处理。 # 并行场景： # 10:00 - 你私聊她："今天天气真好" # 10:01 - 同事在群里发了个工作问题 # 10:01 - AI-B 私聊她："你在吗" # 她的处理： # → 收到你的消息 → 使用「你的关系档案」→ 亲近语气回复 # → 收到群消息 → GroupBehavior判断 → 可能回/不回 # → 收到AI-B → 使用「AI-B的关系档案」→ 友好但有距离 # 每个窗口的上下文是独立的： # ChatContextManager 为每个 chat_id 维护独立的 message_history # 她不会把群里的话题带到你的私聊里（除非自然想到）

❓ 常见问题

Q: 她在群里会不会说太多/太少？

▼

GroupBehaviorEngine 控制回复概率。你可以通过调整 persona.personality.extraversion 来改变她在群里的活跃度。外向值0.8的她会比0.3的她在群里发言多很多。你也可以直接修改 should_respond() 中的概率公式。

Q: 两个AI聊天会不会无限循环？

▼

会的。建议在群聊场景中添加anti-loop机制：检测到对方是bot时降低回复概率（如×0.3）、限制每5分钟最多回复同一bot 2次、或者在回复之间加入更长的随机延迟。在 handle_message 中可以通过 Telegram 的 is_bot 字段判断。

Q: LLM 费用大概多少？

▼

每次回复调用1次 expression LLM（prompt约2000 token + 回复约200 token）。如果使用 GPT-4o-mini，大约 $0.001/次。关系深度反思（每20次对话1次）和记忆整合（每6小时1次）各额外调用1次 utility LLM。日常聊天量约100条/天的话，月费用大约 $3-5（GPT-4o-mini）或 $15-25（GPT-4o）。

Q: 记忆数据库会不会越来越大？

▼

forget_and_distort() 会定期清理低权重记忆（accuracy < 0.15 的自动删除）。情感权重高的记忆保留更久（就像人对情感强烈的事印象更深）。实际使用中，大约每3个月会稳定在 2000-5000 条记忆左右。如果需要更激进的清理，可以调低遗忘系数。

Q: 可以不用 Telegram，换成其他平台吗？

▼

可以。Orchestrator 是平台无关的。bot.py 只是 Telegram 的适配层。你可以写一个 discord_bot.py、wechat_bot.py、或者 web_api.py，只要调用 orchestrator.handle_message() 并传入正确的参数即可。核心的关系/记忆/情绪系统完全不依赖 Telegram。

Q: 她知道自己是AI吗？

▼

在prompt中明确指示"不要说作为AI"。但这不是万无一失的——如果用户直接问"你是AI吗"，LLM 可能会出戏。建议在 persona 设定中加入对这种问题的预设反应（比如"开玩笑的吧？""你说呢？"等回避策略）。
